The project operates on collections. However all collections in this project has to be appeneded with 'tgt_graph_<collection_name>.  In the api's where ever collection name is sent it will not contain the 'tgt' prefix. So we need a middleware which , for each api, that recieves collection name , appends tgt_graph as prefix, and for all api's that respond collection name, or a list of collection names etc we need to remove the prefix 'tgt_graph' from the collection name in response. In essence, to the world outside this project collection name is '<collection_name>' and within this project the collection name that will be used will be 'tgt_graph_<collection_name>'. So assess the entire  codebase and make all necessary changes to ensure this distinction is always maintained. Lets be thourough and precise in our changes.
